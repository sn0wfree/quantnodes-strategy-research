## 2. External references (read in full where applicable)

### 2.1 Red Hat ‚Äî "Designing the User Experience of Command Line Tools" (2024 ed.)

> URL: https://access.redhat.com/articles/linux-cli-design

**Summary.** RFC-style guidance document covering visual hierarchy, error
recovery, and progressive disclosure in TTY tools. Recommends (a) default
values inline so users only type changes; (b) non-zero exit only on hard
failures, never on Cancel; (c) keep destructive ops out of the hot path.

**Applied.** Our `_maybe_run_onboarding` follows (a) ‚Äî provider/model/timeout
all ship with sensible defaults (Claude Sonnet, 300s) and the wizard only
needs Y/N + secret input. Per (b) `run_onboarding()` returns `None` (soft
cancel) ‚Üí CLI exits 0, while a crash inside `_finalize` ‚Üí non-zero. Per (c)
the `.env.partial ‚Üí os.replace` cycle keeps a corrupt `.env` from leaking
into the hot path of the next invocation.

### 2.2 `python-prompt_toolkit` 3.0 ‚Äî Application / HSplit / KeyBindings chapters

> URL: https://python-prompt-toolkit.readthedocs.io/

**Summary.** Documents the `Application` lifecycle: `prompt`, `key_bindings`,
`layout`, and the 3 sentinel-return idiom (`Esc`, `Left`, `Ctrl+C`). Native
vertical-scroll selector uses `HSplit([Window(FormattedTextControl(...))])`
with an `index` cursor managed by `@kb.add("up"/"down")` handlers; `Esc`
returns a sentinel object (e.g. `BACK`) the dispatcher treats as "step ‚àí1".

**Applied.** `cli/onboard.py` (already merged earlier this session) ships a
`run_onboarding` whose internal loop maintains `i = 0; while i < len(steps)`,
decrements on `BACK`, breaks clean on `CANCEL`. No new code needed for
v0.5.0 ‚Äî the survey confirms the existing implementation matches the
reference's idioms exactly.

### 2.3 `motdotla/dotenv` ‚Äî README, esp. the candidate-path resolution section

> URL: https://github.com/motdotla/dotenv/blob/main/README.md

**Summary.** Documents the order of precedence for `.env` discovery:
process env vars, project-local `.env`, `$HOME/.env`. Critically, recommends
`os.replace` over `os.rename` (latter fails across filesystems on POSIX when
tmpdir ‚âÝ targetdir). Also recommends `chmod 0o600` for credential files.

**Applied.** `cli/_auto_onboard.py::_maybe_run_onboarding` mirrors the
3-candidate precedence: `HOME/.env` ‚Üí `<pkg>/.env` ‚Üí `<cwd>/.env`. The
`_finalize` helper in `cli/onboard.py:128` uses `mkstemp(prefix=".env.",
dir=env_dir)` + `os.replace` ‚Äî same filesystem, so atomic. `chmod 0o600`
lives in both `_save_partial` and `_finalize`.

## 3. Findings applied to v0.5.0 design

| Finding | Where it landed |
|---|---|
| 3-candidate env probe, prefer HOME | `cli/_auto_onboard.py::_PROJECT_ENV_PATH`, `_CWD_ENV_PATH` |
| Atomic write, never `os.rename` cross-fs | already correct in `cli/onboard.py:_finalize` |
| `chmod 0o600` best-effort | already correct in both helpers |
| Cancel is silent exit 0 | `run_onboarding` returns `None` ‚Üí CLI exits 0 |
| `BACK` decrements index | already correct in onboard's `while i < len(steps)` loop |
| Destructive ops out of hot path | `cmd_init` no longer touches cwd; `cwd/.env` is **advisory** only |
| Single source of truth for wizard | not split Rich vs prompt_toolkit like vibe-trading legacy |

## 4. Migrator's note (legacy users)

`~/.strategy-research/.env` ‚Üí `~/.quantnodes/strategy_research/.env`. Migration
