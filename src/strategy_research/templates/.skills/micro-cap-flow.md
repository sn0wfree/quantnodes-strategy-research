# Micro-Cap Flow

微盘股(< 50亿市值) 资金面驱动极强, 但也最容易受操纵。本 skill 教如何利用资金流数据但不被坑。

## 数据源

| 来源 | 内容 | 实时性 |
|------|------|--------|
| 龙虎榜 (Tushare) | 异动股买卖前 5 营业部 | 日 |
| 大宗交易 (Tushare) | 折溢价成交 | 日 |
| 北向资金 | 沪深港通 | 日/分钟 |
| 融资融券 | 两融余额 | 日 |
| 涨停板 | 涨停股池 | 实时 |
| 盘后成交回报 | 主力净流入 | 日 |

## 龙虎榜解读

### 触发条件 (任一)
- 日涨幅 ±7% (创业板/科创板 ±20%)
- 日换手率 > 20%
- 日振幅 > 15%
- 连续 3 个交易日涨幅 ±20%

### 席位类型识别
| 营业部类型 | 特征 | 策略价值 |
|-----------|------|---------|
| 知名游资 (如赵老哥、孙哥) | 涨停封板快 | 跟随 |
| 机构席位 | 出现概率低, 量大 | 中长线 |
| 量化席位 | 频繁出现, 多股票 | 短期动量 |
| 散户大本营 | 营业部在拉萨等地 | 反向指标 |

### 信号合成
```python
# 多个游资席位同向买入 → 强信号
# 机构席位大额买入 + 游资跟风 → 中长线
# 游资大额买入但机构缺席 → 短线博弈
```

## 大宗交易

### 数据字段
- 成交价 (vs 收盘价)
- 折溢价 (-10% ~ +10%)
- 买方/卖方营业部
- 成交金额 (> 1000 万为大单)

### 信号
- **折价大额**: 可能是机构出货 / 股东减持, 短期看空
- **溢价大额**: 战略投资者接手, 长期看多
- **频繁对倒**: 同营业部买卖, 可能是利益输送

## 微盘股涨停接力策略

### Step 1: 筛选
```python
# 涨停 + 流通市值 < 50亿 + 首次涨停
filtered = stocks[
    (stocks.pct_change > 0.10) &  # 涨停
    (stocks.limit == 'U') &        # 涨停
    (stocks.float_cap < 5e9) &     # 5亿流通市值
    (stocks.first_limit_today)      # 首次涨停 (不是连板)
]
```

### Step 2: 看接力意愿
- 封单金额 / 当日成交额 > 30%: 强封板
- 封单金额 < 5%: 弱封板, 可能开板
- 开板次数 > 3: 弱

### Step 3: 次日表现预测
| 封板强度 | 游资席位 | 题材热度 | 次日预期 |
|---------|---------|---------|---------|
| 强 | 知名游资 | 主线 | 高开高走 |
| 强 | 量化 | 非主线 | 高开震荡 |
| 弱 | 未知 | 非主线 | 低开 |
| 弱 | 知名游资 | 主线 | 平开震荡 |

## 风险点

1. **操纵风险**: 微盘股最易被操纵, 龙虎榜数据本身可能造假
2. **流动性**: 大资金进出冲击成本 > 1%
3. **退市风险**: 微盘股退市比例 > 5%/年
4. **停牌**: 异动后可能被特停
5. **回测陷阱**: 用龙虎榜回测会高估收益 (幸存者偏差 + 信号衰减)

## 实施流程

### Step 1: 数据准备
```bash
quantnodes-research import --source tushare \
  --table top_list \
  --start 2020-01-01
```

### Step 2: 筛选规则
- 涨停股 + 流通市值 < 50 亿
- 剔除 ST, 退市, 停牌

### Step 3: 信号
```python
# 当日封板强度 + 次日开盘相对强弱
signal = (
    0.5 * standardized封单强度 +
    0.3 * standardized游资席位score +
    0.2 * standardized题材热度
)
```

### Step 4: 持仓
- 单只最大仓位: 2% (避免单股黑天鹅)
- 持有期: 1-5 日
- 止损: -5% 严格

## 验证指标

| 指标 | 健康阈值 |
|------|---------|
| 次日胜率 | > 55% |

| 最大单股回撤 | < -8% |
| 周转率 | < 10%/周 |
| 与基准相关 | < 0.5 |
